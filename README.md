## What This Is

This isn't a portfolio template with my name pasted on it. It's a **digital garden and field journal** — a single, structured system for documenting how I think, build, and learn in public, as a second-year engineering student at BITS Pilani, Pilani Campus.

Most student portfolios are static resumes dressed up in a nicer font. This one is designed to be a **living record**: it updates as I do, section by section, project by project.

The thesis behind it: **curiosity compounds faster than credentials.** This site is the infrastructure for testing that thesis, in public.

> **Status:** This is an early scaffold. The structure, sections, and copy are currently a starting template and do not yet reflect finished, personal content — projects, writing, journey entries, and resources will be filled in and revised as the site develops.

## Tech Stack

- **Framework:** [Astro](https://astro.build) (via the [AstroWind](https://github.com/onwidget/astrowind) template, heavily customized)
- **Styling:** Tailwind CSS
- **Content Management:** [Decap CMS](https://decapcms.org) with PKCE-based OAuth for git-backed editing
- **Hosting:** Vercel, with view-transitions enabled for smooth page navigation
- **Content model:** Fully static-generated, git-as-database — every project, journal entry is a version-controlled markdown file

## Running Locally

```bash
# Clone the repository
git clone https://github.com/<your-username>/digital-hq.git
cd digital-hq

# Install dependencies
npm install

# Start the dev server
npm run dev

# Build for production
npm run build
```

Content (projects, writing, journey entries) is managed through **Decap CMS** at `/admin`, so non-code updates ship without touching a text editor.

## Philosophy

- **Curiosity over comfort** — I choose the harder path when it teaches me more.
- **Build in public** — ideas are cheap; shipping and documenting the process is everything.
- **Depth over breadth** — going deep on a few things beats skimming everything.
- **Genuine, always** — no personas, no performing for an audience. What you see is what's actually happening.

I'd rather understand something deeply and build with it than collect credentials and call that understanding. This site is the proof of that, collected in public.
